private List<List<String>> partition(List<String> source,
                                     int amountOfParts) {
    List<List<String>> parts = new ArrayList<>(amountOfParts);
    for (int partIndex = 0; partIndex < amountOfParts; partIndex++) {
        Range range = calculateRange(partIndex, amountOfParts);

        List<String> part = source.subList(range.start, range.end);
        parts.add(part);
    }

    return parts;
}

Rebasing

Opdracht (+- 15 minuten)

Part 1

  1. Creer een nieuwe branch vanaf master
  2. Maak op de branch een aanpassing in bestand1.txt en commit dit
  3. Ga nu terug naar de master branch
  4. Maak een aanpassing in bestand1.txt waar een merge conflict zou moeten onstaan met de wijziging op je branch en commit dit
  5. Merge nu je wijzigingen naar master en los de merge conflicten op.
  6. Bekijk de graph en onthou wat je ziet.

Part 2

  1. Reset je repository naar de laatste commit voor de merge (NEW)
  2. Rebase de nieuwe branch op master
  3. Merge de branches en bekijk de graph
  4. Wat is het verschil?


Tijd over? Probeer de rebase eens via een interactive rebase te doen!

Graph: git log --graph --oneline --all

Opdracht (+- 10 minuten)

  1. Ga naar een nieuwe directory en clone de repository waar bestand1.txt in staat
  2. Commit hier een aantal wijzigingen op de master branch
  3. Push deze wijzigingen naar de remote
  4. Maak vervolgens nog een wijziging en commit deze. Push de wijziging nog niet!

  6. Ga terug naar de lokaal uitgecheckte repository en probeer je wijzigingen te pushen.
